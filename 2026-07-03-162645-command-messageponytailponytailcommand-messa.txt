     Offline-first startup: show local vault before sync

     Context

     Sync works, but on app start the UI is empty until the startup Nextcloud
     sync finishes. The local vault files are on disk the whole time — nothing
     is lost. The bug is ordering in _open() (lib/app_mobile.dart:88-131): when
     cloud is configured, it awaits NextcloudSync(cfg).sync(v, trigger: 
     'startup') at line 101 before the local scan (todayNote, rebuildIndex,
     _loadSource) and the populating setState at lines 108–123. index and note
     stay null for the whole sync, so the list and editor render empty. Sync
     does not wipe anything; there is no cache-dir mismatch.

     Fix (one function, lib/app_mobile.dart _open())

     Reorder _open() to be offline-first:

     1. NextcloudConfig.load() + Vault.openDefault() (unchanged).
     2. Immediately run the local load: todayNote, rebuildIndex,
     loadTagRegistry, loadFileRegistry, validatePkms, _loadSource, then the
     populating setState (also sets vault, cloud so the sync helpers can see
     them). Status: 'Vault: ${v.root.path} · ${report.summary()}'.
     3. If cfg != null && cfg.isReady: unawaited(_syncNow(trigger: 'startup'))
     and _startCloudPolling().

     Delete the inline awaited sync block (old lines 93–107) and the
     _queueCloudSync() call at startup — _syncNow already sets the syncing
     flag/status, re-reads the current note after sync, rebuilds the index, and
     setStates (lib/app_mobile.dart:297-332), so the UI updates itself when sync
     lands. Calling _syncNow directly also removes the old double-sync (awaited
     startup sync followed 2s later by _queueCloudSync's autosave-trigger
     sync).

     Verification

     - flutter analyze clean.
     - Run the app with Nextcloud configured but network slow/off: pages list
     and today's note appear immediately from local disk, status shows the vault
     path, sync runs in background and updates status when done (or shows 'Sync
     failed: …' without blanking content).
     - Run once with no cloud config: unchanged behavior.

⏺ Fixed. The cache was never lost — _open() just blocked on the startup
  Nextcloud sync before ever reading local disk, so index/note stayed null and
  the UI rendered empty until sync returned.

  The fix in lib/app_mobile.dart _open(): local vault now loads and renders
  immediately (today's note, index, registries), then sync fires in the
  background via unawaited(_syncNow(trigger: 'startup')) — which already
  re-reads the note, rebuilds the index, and updates the UI when it lands. Also
  dropped the old _queueCloudSync() at startup, which was scheduling a redundant
  second sync 2 seconds after the first.

  flutter analyze clean, all 26 tests pass. Offline start now shows content
  instantly; a failed sync just sets the status line instead of blanking the
  app.
