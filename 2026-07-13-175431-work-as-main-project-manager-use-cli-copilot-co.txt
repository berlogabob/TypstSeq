     TyLog audit remediation — P0 (data safety) + P1 (the 6 UX gaps)

     Context

     Full 3-agent audit of TyLog (Flutter local-first Typst journal, lib/ +
     packages/typst_flutter) against the user's 6 concerns. Verdict: SAF storage
     and the Nextcloud sync engine are fundamentally sound; PKMS is mostly
     wired. The real problems are 3 data-safety bugs and a today-anchored,
     checkbox-less UX. User decisions: remove quick note, Logseq-style calendar 
     integrated with the PKMS index (month grid with entry markers, navigable
     date refs, linked references per day), scope P0+P1.

     P0 — data safety

     1. .backup orphans pollute Nextcloud

     Interrupted SAF atomic replace leaves .<name>.tylog-<nanos>.backup in the
     vault (android/app/src/main/kotlin/org/tylog/tylog/SafBridge.kt:376-383);
     isSyncInternalPath (lib/nextcloud_sync.dart:1088) doesn't filter it, so it
     uploads.
     - Add .backup (basename starts with . and ends .tylog-*.backup) to
     isSyncInternalPath.
     - On vault open, delete stale .backup orphans (sweep in the existing
     startup scan path, lib/app_mobile.dart _openVault).
     - Test: extend test/nextcloud_sync_test.dart — a .backup file in the vault
     must not upload.

     2. Download integrity

     _downloadStorage/_download (lib/nextcloud_sync.dart:723-782) commit
     whatever bytes arrive; chunked truncation is silent. Uploads already verify
     x-hash-sha256 (:699-702).
     - After download, verify received length against Content-Length when
     present, and verify sha256 against the server hash header when present
     (same header family the upload path already uses); on mismatch, discard
     temp and abort that file — next poll retries.
     - Test: mock chunked truncated response → file must NOT be committed.

     3. Quick-capture lost-update race → resolved by deletion (P1 item 4 removes
     quick note entirely). While there, add the missing dirty-guard to 
     _showToday/_selectDestination case 0 (lib/app_mobile.dart:1457, 2108) 
     matching _showTasks (:2103).

     P1 — the six UX gaps

     4. Remove quick note (user concern #4)

     - Delete the "Quick note…" TextField + onCapture wiring in _TodayView
     (lib/app_mobile.dart:3480-3497) and _quickCapture (:2073-2092). Keep "Open
     today's journal" button.

     5. Launch into today's file editor (concern #1/#4 design intent)

     - Startup currently lands on the Today dashboard: mode field init 'today'
     (lib/app_mobile.dart:148) + startup guard (:332). Today's file is already
     loaded (:309-317) — just start in mode='normal' (the journal editor). Today
     dashboard stays reachable via nav.

     6. Date navigation, Logseq-style: dates are PKMS nodes (concern #3)

     The calendar must be wired into the PKMS index, not a bare dialog. All data
     already exists in VaultIndex.calendar (lib/models.dart:312-341 — daily
     notes, date-refs, task due dates keyed by ISO day).

     - Vault.dailyNote(DateTime day): generalize todayNote()
     (lib/vault.dart:107-125); reuse _day formatting (lib/vault.dart:285).
     todayNote calls it with now.
     - Month-grid picker with entry markers: small custom month grid (Flutter's
     showDatePicker can't mark days). One stateless widget (~100 lines): month
     view, dot on any day present in index.calendar (distinct style for
     has-daily-note vs has-refs/tasks-only), tap → open/create dailyNote(day),
     chevrons for month paging. Opened from the journal header date button.
     Replace the flat Library "Calendar" agenda tab
     (lib/app_mobile.dart:3652-3687) with this grid + the day's items listed
     below it.
     - ◀ ▶ day arrows in the journal editor header when the open note is a daily
     file → dailyNote(current ± 1 day).
     - Tappable date refs: a #tylog.date-ref atom in the block view
     (lib/rich_editor.dart date atom) navigates to that day's journal page on
     tap — Logseq behavior. (Currently dates are only insertable, not
     navigable.)
     - Linked references on a day's page: when a daily file is open, show the
     items referencing that date — date-refs and due tasks from
     index.calendar[day] plus backlinks to the daily note — reusing the existing
     backlinks side panel (lib/app_mobile.dart:3143-3162) and the Today
     "mentions" pattern (:3543-3554).
     - Tests: unit test dailyNote path for arbitrary dates; unit test that the
     grid's marker source maps index.calendar days correctly.

     7. Tappable task checkboxes in the block view (concern #6)

     - lib/rich_editor.dart:1045-1061 renders #tylog.task(...) as an opaque
     protected chip. Render it instead as a checkbox row (status from parsed
     status: field); tap → existing replaceTaskStatus (lib/scanner.dart:615-630)
     → save. Same write-back the Tasks list already uses (_setTaskStatus,
     lib/app_mobile.dart:960-976) — reuse it.
     - Test: widget/unit test — toggle writes correct status: back into source
     (round-trip).

     8. Stop collapsing paragraphs with unknown inline Typst (concern #6)

     - _parseInline returns null on any unrecognized #
     (lib/rich_editor.dart:1103, 1183), collapsing the whole paragraph into a
     hidden "Custom Typst" chip. Change: parse the unknown #call(...)/#call[..]
     as an inline protected atom (balanced-paren scan already exists in
     lib/controlled_editor.dart:99-109) and keep surrounding prose editable.
     Only fall back to the whole-block chip if the inline call itself can't be
     delimited.
     - Also accept #link("url") without a label in the atom regex
     (lib/rich_editor.dart:1167-1169) — fixes the layer-1/layer-2 grammar
     mismatch.
     - The existing toSource() re-parse guard (lib/rich_editor.dart:391-413)
     protects against corruption; keep it.
     - Test: paragraph with #footnote[x] and one with #link("https://x") stay
     editable text + atom, and round-trip byte-identical.

     9. Citations render (concern #5)

     - Preview: in _typstFiles() (lib/app_mobile.dart:675-680), when
     bibliography.yml exists, include it and append
     #bibliography("/bibliography.yml") to the preview wrapper (only when the
     note contains a citation, to avoid empty-bib errors).
     - Reports: same #bibliography line in generateReportSource
     (lib/report.dart:71-77) when citations present (exportReportPdfStorage
     already injects all vault files).
     - Test: preview compile of sample_vault/articles/Smith 2026.typ succeeds.
